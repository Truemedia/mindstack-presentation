Why is a new framework solution needed?

ğŸ›  STRUCTURE
Most framework or api options leave very little in terms of ideas on how to structure your application. Improvising in this area can be daunting and quickly come back to bite you if you opt for the wrong strategy which essentially becomes a game of luckily fishing for useful libraries to get one thing done in the only way you can think how.
ğŸ—„  AMOUNT OF CODE
To make any kind of skill with a reasonable amount of functionality and great user experience for multiple api's without a framework would be a massively daunting and time consuming experience. Why would you want that when boilerplate can save you from spaghetti code and hours of your life back?
ğŸ“„  MARKUP
Speech assistants like Alexa use SSML with some custom tags where as text chat services like Slack use custom markdown. There isn't any library currently to write a template that works for all and very few bot frameworks use templating libraries.
ğŸ“¡  SERVICE API'S
There is no consolidated API for the services I want to use. Why write code more than once when it's doing the same thing on a different services?
ğŸ¤“  UNINTELLIGENT
Most chat bot architectures are still using conventions which are outdated and clunky, where as people want natural conversations with smart AI. Even voice assistants are lacking in terms of the leg work needed to recognise a wide spectrum of conversation scenarios which people would want to essentially to perform the same task.
ğŸ˜  LACK OF PERSONALITY
It's nice to have a conversation with something that seems somewhat intelligent and has a personality you can connect with on a personal level. Generic and stale language can make the experience boring and feel like a chore more than a nice thing to have.

Solution?

Mindstack: A powerful and flexible bot framework for building bots without restrictions

Github - Truemedia/mindstack NPM - mindstack

  • Project was started on October 14th 2018 (Just late last year)
  • < 100mb in size when installed
  • Can build a bot in < 20 lines of code (*.js)
  • Example code, adapters, and skills easy to find and add
  • Generator/s available
  • Tests fully implemented
