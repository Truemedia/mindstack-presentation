Why is a new framework solution needed?

MARKUP
Speech assistants like Alexa use SSML with some custom tags where as text chat services like Slack use custom markdown. There isn't any library currently to write a template that works for all and very few bot frameworks use templating libraries.
SERVICE API'S
There is no consolidated API for the services I want to use. Why write code more than once when it's doing the same thing on a different services?
UNINTELLIGENT
Most chat bot architectures are still using conventions which are outdated and clunky, where as people want natural conversations with smart AI. Even voice assistants are lacking in terms of the leg work needed to recognise a wide spectrum of conversation scenarios which people would want to essentially to perform the same task.
LACK OF PERSONALITY
It's nice to have a conversation with something that seems somewhat intelligent and has a personality you can connect with on a personal level. Generic and stale language can make the experience boring and feel like a chore more than a nice thing to have.
STRUCTURE
Most framework or api options leave very little in terms of ideas on how to structure your application. Improvising in this area can be daunting and quickly come back to bite you if you opt for the wrong strategy which essentially becomes a game of luckily fishing for useful libraries to get one thing done in the only way you can think how.
AMOUNT OF CODE
To make any kind of skill with a reasonable amount of functionality and great user experience for multiple api's without a framework would be a massively daunting and time consuming experience. Why would you want that when boilerplate can save you from spaghetti code and hours of your life back?

Solution?

New bot framework

Lowbot: low implementation bot

https://github.com/Truemedia/lowbot

  • Project was started on October 14th 2018 (Just 2 months ago)
  • < 13kb in size when installed
  • Can build a bot in < 20 lines of code (*.js)
  • Example code, adapters, and skills easy to find and add
  • Generator/s available
  • Tests fully implemented

Architecture

Utterances
Phrases/sentences to match user speech to an intent
Intent
A predefined action that the bot can respond to (using utterance)
Slot
A dynamic piece of information derived inside an utterance
Lexicon
Dictionary of words/phrases corresponding to meaning such a noun, verb, synonym etc
Resolvers
Functions executed when an intent/intents have been recognised
Templates
SSML, MD, HTML, Text files used for templating content in responses
Adapters
Object used to integrate bot into specific service/network (discord, alexa, etc)
Skillsets
Skills to give your bot functionality
Persona
Object to dictate the personality of a bot, using descriptive parameters
Data services
External services for providing data and resources inside intent handlers
POD service
Personal online datastore, A centralised service for grabbing personal details of someone from a shared datastore.
Env file
Set credentials/environment variables for adapters, data services, and pod services

Adapters

Adapters allow you to connect your bot to different services as a messaging/voice medium.

Currently supported adapters include:

  • Alexa
  • Discord
  • Email
  • Slack
  • SMS
  • Terminal

Skillsets

Ready to use now on any bot instances are a group of skills built to meet common desirable use cases.

Currently we have built:

Basic skill
Greetings and goodbyes, time and date, on and off.
Announcement skill
Post content to multiple services simultaneously and use reminders
Fact skill
Ask for very specific facts (example is on planets)
Resource skill
CRUDL functionality for specified resource/s (example is on gardening)
Story skill
Tell an interactive story (example is basic RPG)
Store skill
Purchase virtual or real products/services (example is bar and clothing store)
IOT skill
Interact with a basic arduino component

Personality

Choose from different personalities or create your own by tweaking an existing one.

Made from relatively well known AI instances of the past, remade for the current era.

  • Paper clip (Clippy)
  • Butler (Ask Jeeves)

Languages

Support any language in existence to target a local or worldwide audience.

For novelty purposes even target ones that are very niche or fictional.

Currently supported languages:

  • English (en_GB)
  • Japanese (ja_JP)

Feel free to contribute if you want to diversify what is on offer.
